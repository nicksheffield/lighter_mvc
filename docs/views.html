Views

app/views/

Views are how you render the data of your application onto the browser.

They have access to all static libraries that have been loaded.

Views are also able to access any php variables that you send along when you load the view.

To load a view from a controller, you use the Loader library. eg.

Load::view('header');
Load::view('homepage');
Load::view('footer');

When you load views with the Loader library, you don't need to specify the .php file extension, as all views are required to be php files anyway.

To send php variables to a view, send them in an associative array through the second parameter of the Load::view() function. eg.

Controller code


$data['title'] = 'Hello World';
$data['content'] = 'Yo waddup';

Load::view('homepage', $data);

View code

<h1><?= $title ></h1>
<p><?= $content ></p>
