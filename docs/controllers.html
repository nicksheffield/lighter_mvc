Controllers

app/controllers/

Controllers are php classes that define sets of related pages.

They are defined as a class that extends the Controller class, and have methods that act as pages.

They relate directly to the request URL. If the user requests the page user/login the User controller is loaded, and the login method is run. The User controller may have other pages defined within it, such as logout, register, edit or delete.

An example of a simple controller:

<?php

class Site extends Controller{

	function index(){
		echo 'Hello World';
	}

}

When the user requests http://example.com/site/index, http://example.com/site or even just http://example.com if site is set as the default controller, 'Hello World' would be displayed.

If you need to make a method inside a controller that is blocked from being accessed via the url, you can do so by adding an underscore to the beginning of the method name. eg.

<?php

class Site extends Controller{

	function index(){
		echo 'Hello World';
	}

	function _secret_function(){
		echo 'This can never be seen in a browser';
	}

}
